# AI 제품·서비스 확인제도 — 기술심사 관리 시스템 (TTA 내부)

한국정보통신기술협회(TTA)가 **심사기관**으로서 수행하는 「인공지능 제품·서비스 확인제도」의 기술심사 업무를 관리하는 Google Apps Script 기반 내부 시스템입니다.

> **역할 구분**
> - **KOSA (확인기관)** — 접수 · 서류심사 · 결정 · 확인서 발급, 전체 시스템 소유
> - **TTA (심사기관)** — 기술심사만 수행. KOSA와 완전히 독립된 시스템 운영
>
> TTA는 KOSA로부터 신청 데이터를 **엑셀로 인수**하고(향후 API 예정), 심사 결과는 **결과 목록만** KOSA에 통보합니다. 심사 증빙은 TTA가 자체 보관(Drive → NAS)하며 KOSA로 넘기지 않습니다.

---

## 구성 파일

| 파일 | 설명 |
|---|---|
| `AI심사관리_AppScript.gs` | 메인 로직 (파싱·시트관리·보고서·웹앱 백엔드) |
| `심사폼.html` | 심사원용 웹 심사 폼 (Apps Script HTML 파일명 = `심사폼`) |

## 시트 구조

`초기설정실행` 실행 시 7개 시트가 자동 생성되며, **일정관리 시트가 맨 앞(메인)** 에 배치됩니다.

| 시트 | 역할 | 원본/참조 |
|---|---|---|
| **일정관리** | 메인 워크플로우 허브. 스케줄·배정·상태 관리, 심사폼 실행 | 하이브리드 (아래 참조) |
| 접수대장 | 신청 원본 데이터 전체 (34컬럼) | 원본 |
| 인공지능제품모델 | 모델명·세부품명번호·물품식별번호 (반복행) | 원본 |
| 인공지능기능상세 | 기능별 업무·구현방식 상세 (22컬럼, 반복행) | 원본 |
| 인공지능심사체크결과 | 심사폼 항목 판정 결과 | 심사 산출 |
| 인공지능기능확인 | 심사폼 기능명세 표의 확인·비고 | 심사 산출 |
| 파싱로그 | 엑셀 파싱 이력 | 로그 |
| 컬럼매핑 | 엑셀 헤더 별칭 관리 (코드 수정 없이 매핑 변경) | 메타 |

### 일정관리 시트의 하이브리드 싱크

- **접수대장이 원본** (VLOOKUP 자동 참조): 기업명·연락처·제품명·개요·제공형태·제품분류·적용목적/범위·명세서작성방식·보유인증·인공지능기능수 → 접수대장을 고치면 자동 반영
- **일정관리가 원본** (직접 편집): 상태·담당심사원
- **자동 계산**: 순번, 심사링크(HYPERLINK), 마감예정일(=신청일+15일), 기타제출서류여부

> VLOOKUP 키는 **접수번호(A열 아님, 헤더 기준)** 이므로 접수대장 행이 정렬·이동돼도 값이 따라갑니다.

## 설치 및 배포

1. Google Sheets 새 스프레드시트 생성
2. 확장 프로그램 → Apps Script → `AI심사관리_AppScript.gs` 전체 붙여넣기
3. HTML 파일 추가 (파일명 정확히 **`심사폼`**) → `심사폼.html` 내용 붙여넣기
4. 저장 후 스프레드시트 새로고침 → 상단 **🔍 인공지능심사관리** 메뉴 확인
5. 메뉴 → **⚙️ 초기 설정 (최초 1회)** 실행 → 시트 자동 생성
6. 웹앱 배포: 배포 → 새 배포 → 유형 "웹 앱" → 실행 계정/접근 권한 설정 → 배포
7. 발급된 `/exec` URL을 코드 상단 `CONFIG.심사폼URL`에 붙여넣기
8. 메뉴 → **🔗 심사 링크 채우기** 실행 → 일정관리 각 행에 심사폼 링크 생성

> **재배포 주의**: 코드 수정 후에는 "새 배포"가 아니라 **배포 관리 → 편집(연필) → 새 버전 → 배포** 로 갱신해야 URL이 유지됩니다. 새 배포를 만들면 URL이 바뀌어 링크를 다시 채워야 합니다.

## CONFIG 설정

코드 상단 `CONFIG` 객체에서 설정합니다.

| 키 | 설명 |
|---|---|
| `담당자목록` | 심사원 자동 배분 대상 |
| `기본심사기간` | 마감일 계산 기준일수 |
| `알림이메일` | 신규 접수 시 알림 발송 대상 |
| `드라이브폴더ID` | 증빙·보고서 저장 폴더 |
| `심사폼URL` | 웹앱 배포 `/exec` URL |
| `관리자이메일` | 관리자 전용 메뉴를 볼 수 있는 계정 (비우면 소유자만) |
| `보고서.*` | 보고서 표지·문서번호·심사근거 등 |

## 권한 (메뉴 분기)

`onOpen`이 실행 계정을 확인해 메뉴를 나눕니다.

- **심사원 공통**: 심사 폼 열기 · 기술심사보고서 생성 · 보고서 3종 산출
- **관리자 전용**: 심사 링크 채우기 · 엑셀 등록 · 초기 설정 · 헤더 재설정 · 시트 재생성 · Chat 알림 테스트

관리자 판정: `CONFIG.관리자이메일` 목록 우선, 비어 있으면 스프레드시트 소유자.

> ⚠️ **한계**: 이는 메뉴 숨김(편의)일 뿐 완전한 보안이 아닙니다. 심사원이 Apps Script 편집기에 접근하면 함수를 직접 실행할 수 있으므로, 실질 차단은 **스프레드시트 공유 권한을 편집자로만 제한**하는 것과 병행해야 합니다.

## 엑셀 인수 → 등록 흐름

1. KOSA로부터 받은 엑셀을 Drive에 업로드
2. 메뉴 → **📁 엑셀 파일 등록 (파싱)** → 파일 ID/URL 입력
3. 3개 탭 자동 인식: `접수현황`(기본정보) / `제품모델` / `인공지능기능상세`
4. 접수번호 기준으로 접수대장·제품모델·기능상세에 등록, 일정관리에 행 추가
   - 같은 접수번호 = 갱신(update) / 새 접수번호 = 추가(append)

### 컬럼매핑 (코드 수정 없이 헤더 대응)

KOSA 엑셀의 실제 컬럼명이 바뀌면 **`컬럼매핑` 시트**의 별칭 칸만 수정하면 됩니다. (A열=내부키, B열=쉼표 구분 별칭). 파서는 완전일치 → 부분일치 순으로 매칭합니다.

## 심사 흐름

1. 일정관리 시트에서 대상 행의 **심사링크** 클릭 → 심사폼 열림
2. 심사폼에서:
   - **인공지능 기능 명세 확인 표**: 기능별 ID(=증빙 캡처 파일명)·확인 체크박스·비고, 마지막 "기타" 행
   - **심사 항목 판정**: 공식 확인 기준 7항목 판정(적합/보완/부적합) + 사유
3. **완료** 클릭 → 체크결과 저장 → 기능확인 저장 → 증적 3종 산출(PDF·DOCX·JSON)

### 확인 기준 (가이드라인 1.1~3.1)

| 번호 | 구분 | 확인 내용 | 방법 |
|---|---|---|---|
| 1.1 | 인공지능처리 연산체계 | 연산체계가 적용되어 있는가 | 서류 |
| 1.2 | 〃 | 학습·추론 등 AI 특성을 가지는가 | 서류·현장 |
| 1.3 | 〃 | 유의미한 결과를 출력하는가 | 서류·현장 |
| 2.1 | 인공지능기능 | 구조도상 위치가 명확하고 결합 동작하는가 | 서류 |
| 2.2 | 〃 | 기능·효율성 등에 활용됨이 확인 가능한가 | 서류·현장 |
| 2.3 | 〃 | 주요 기능이 정상 동작하는가 | 현장 |
| 3.1 | 외부 연동 | 호출 로그·API Key 확인 가능한가 | 서류·현장 |

### 구현방식 분류

`A` 자체 학습 모델 · `B` 사전학습+파인튜닝 · `C` 외부 API/모델 활용 · `D` 엣지 디바이스 추론 · `E` 혼합형

## 데이터 무결성 설계

- **모든 시트 쓰기는 헤더 이름 기반** (위치 기반 아님). 심사원이 컬럼을 수동 추가·삽입해도 값이 밀리지 않고, 모르는 컬럼은 빈칸 유지.
- **마이그레이션**: 구버전 시트명·컬럼명 자동 이관 (`헤더마이그레이션`). 이름이 바뀐 컬럼은 기존 값도 새 컬럼으로 복사.
- 위험 작업(`시트재생성_데이터초기화`)은 관리자 메뉴 + 확인 대화상자로 이중 보호.
