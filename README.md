# AI 제품·서비스 확인제도 — 기술심사 관리 시스템 (TTA 내부)

한국정보통신기술협회(TTA)가 **심사기관**으로서 수행하는 「인공지능 제품·서비스 확인제도」의 기술심사 업무를 관리하는 Google Apps Script 기반 내부 시스템입니다.

> **역할 구분**
> - **KOSA (확인기관)** — 접수 · 서류심사 · 결정 · 확인서 발급, 전체 시스템 소유. 신청기업이 작성하는 **입력폼(신청서)** 도 KOSA 시스템에서 운영됩니다.
> - **TTA (심사기관)** — 기술심사만 수행. KOSA와 완전히 독립된 시스템 운영
>
> TTA는 KOSA로부터 신청 데이터를 **엑셀로 인수**하고(향후 API 예정), 심사 결과는 **접수대장의 심사 결과 컬럼**으로 관리하며 KOSA에는 결과만 통보합니다. 심사 증빙(구조도 원본 등)은 TTA가 Google Drive에 접수번호별로 자체 보관합니다.

> 신청서(입력폼) 각 항목의 데이터타입·필수여부·예시는 [입력폼_명세서.md](./입력폼_명세서.md)를 참고하세요.

## 구성 파일

| 파일 | 설명 |
|---|---|
| `Code.js` | 전체 로직 (시트 관리·엑셀 파싱·보고서 생성·알림·웹앱 백엔드 없는 단일 Apps Script) |
| `appsscript.json` | Apps Script 프로젝트 매니페스트 (시간대, 런타임 등) |
| `.clasp.json` | `clasp` 배포용 스크립트 ID 설정 |
| `샘플_신청데이터_*.xlsx` | KOSA 신청 데이터 형식을 흉내 낸 샘플 엑셀 (파싱 테스트용) |

> 과거 존재했던 심사원용 웹 심사 폼(`심사폼.html`, 웹앱 배포, 심사링크)은 제거되었습니다. 현재는 **Google Sheets 메뉴 실행 + 엑셀 인수**만으로 동작합니다.

## 시트 구조

`초기설정실행` 실행 시 시트가 자동 생성되며, **일정관리 시트가 맨 앞(메인)** 에 배치됩니다.

| 시트 | 역할 | 원본/참조 |
|---|---|---|
| **일정관리** | 메인 워크플로우 허브. 스케줄·배정·상태 확인용 최소 필드 뷰 | 하이브리드 (아래 참조) |
| 접수대장 | 신청 건별 기본정보 + 진행상태 전체 데이터베이스 | 원본 |
| 인공지능제품모델 | 모델명·세부품명번호·물품식별번호 (반복행) | 원본 |
| 인공지능기능상세 | 기능별 업무·구현방식(A~E) 세부 정보 (반복행) | 원본 |
| 파싱로그 | 엑셀 파싱 이력 | 로그 |
| 컬럼매핑 | 엑셀 헤더 별칭 관리 (코드 수정 없이 매핑 변경) | 메타 |
| 심사원관리 | 심사원명·이메일·활성여부·배정순서·Chat 사용자 ID 관리 | 운영 설정 |
| 배정알림로그 | 관리자 확정 발송의 접수번호·수신자·결과·실행자 기록 | 로그 |

각 시트가 담는 실제 항목(데이터타입·필수여부·예시)은 [입력폼_명세서.md](./입력폼_명세서.md)에 정리되어 있습니다.

### 일정관리 시트의 하이브리드 싱크

- **접수대장이 원본** (VLOOKUP 자동 참조, 접수번호 기준): 기업명·담당자명·연락처·이메일·소재지·제품명·제품수·개요·제공형태·제품분류·인공지능적용목적/범위·명세서작성방식·보유인증·인공지능기능수 → 접수대장을 고치면 자동 반영
- **일정관리가 원본** (직접 편집): 상태·보완요청일·담당심사원·특이사항
- **자동 계산**: 순번, 마감예정일(=신청일+15 WD), 연장마감일(=보완요청일+30 WD), 기타제출서류여부 — WD 계산 시 주말·대한민국 공휴일 제외

> VLOOKUP 키는 **접수번호(헤더 기준, 열 위치 아님)** 이므로 접수대장 행이 정렬·이동돼도 값이 따라갑니다. 심사원이 시트에 컬럼을 수동으로 추가·삽입해도 이름 기준으로 값이 밀리지 않습니다.

`보완요청일`은 내부 심사원이 캘린더로 입력하고, `연장마감일`은 보완요청일로부터 30 WD 뒤로 자동 계산됩니다. 접수대장과는 동기화하지 않습니다. 보완요청 전에는 `마감예정일`, 보완요청 후에는 `연장마감일`이 만료되면 해당 행을 빨간색으로 표시합니다.

`특이사항`은 담당심사원 오른쪽에 있는 내부 협업용 자유 입력 컬럼입니다. 모든 사용자가 특이사항이나 의견을 자유롭게 작성할 수 있으며 접수대장·보고서와 자동 동기화되지 않습니다. 시트 권한은 Google Sheets의 **시트 및 범위 보호**에서 다른 컬럼을 보호하고 `특이사항` 컬럼만 편집 가능 범위로 제외하여 설정합니다.

## 설치 및 배포

1. Google Sheets 새 스프레드시트 생성
2. 확장 프로그램 → Apps Script → `Code.js` 전체 붙여넣기, `appsscript.json` 매니페스트 반영
3. 저장 후 스프레드시트 새로고침 → 상단 **🔍 인공지능심사관리** 메뉴 확인
4. 메뉴 → **⚙️ 초기 설정·컬럼 갱신** 실행 → 시트 자동 생성 + 누락 컬럼 마이그레이션 + 상태/판정 드롭다운 + 조건부서식 적용
5. `심사원관리` 시트에 실제 심사원명·이메일·활성여부·배정순서를 입력하고, Script Properties에 공용 Chat Webhook을 설정

> `clasp`으로 배포하는 경우 `.clasp.json`의 `scriptId`가 가리키는 Apps Script 프로젝트에 `clasp push`로 반영합니다.

## CONFIG 설정

코드 상단 `CONFIG` 객체에서 설정합니다.

| 키 | 설명 |
|---|---|
| `담당자목록` | `심사원관리` 시트가 비어 있을 때 최초 기본값으로 사용하는 라운드로빈 목록 |
| `기본심사기간` | 접수대장 `심사마감일` 및 Chat/이메일 알림에 실제로 사용되는 일수(기본 14일, 파싱일 기준). **일정관리의 `마감예정일`(신청일+15 WD, 주말·공휴일 제외)과는 별개 계산**이므로 두 값이 다르게 표시될 수 있습니다. 법적 처리기한 판단은 신청일 기준인 일정관리 값을 우선하세요 |
| `드라이브폴더ID` | 증빙 원본·보고서 저장 루트 폴더 ID (비우면 `심사증적_원본보관` 폴더 자동 생성) |
| `챗_공통Webhook` | 관리자가 확정한 선택 행을 묶어 발송할 Google Chat 공용방 Webhook URL |
| `보고서.*` | 기술심사보고서 표지에 들어가는 작성기관·문서번호접두·심사근거·심사방법·보안등급·책임자직위 |
| `관리자이메일` | 관리자 전용 메뉴를 볼 수 있는 계정 목록 (비우면 스프레드시트 소유자만) |

## 권한 (메뉴 분기)

`onOpen`이 실행 계정을 확인해 메뉴를 나눕니다.

- **심사원 공통**: 📑 기술심사보고서 생성 (선택 행)
- **관리자 전용**: 📁 엑셀 파일 등록 · 🔄 선택 행 라운드로빈 추천 · 📨 선택 행 배정 확정·알림 발송 · 🔁 선택 행 재발송 · ⚙️ 초기 설정·컬럼 갱신 · 📅 마감일 갱신 · 💬 Chat 알림 테스트

관리자 판정: `CONFIG.관리자이메일` 목록 우선, 비어 있으면 스프레드시트 소유자.

> ⚠️ **한계**: 이는 메뉴 숨김(편의)일 뿐 완전한 보안이 아닙니다. 심사원이 Apps Script 편집기에 접근하면 함수를 직접 실행할 수 있으므로, 실질 차단은 **스프레드시트 공유 권한을 편집자로만 제한**하는 것과 병행해야 합니다.

## 엑셀 인수 → 등록 흐름

1. KOSA로부터 받은 엑셀을 Drive에 업로드
2. 메뉴 → **📁 엑셀 파일 등록 (파싱)** → 파일 ID 또는 공유 URL 입력 (원본 `.xlsx`만 지원, 구글 시트로 변환된 파일은 거부)
3. 3개 탭 자동 인식: 접수현황(기본정보) / 제품모델 / 인공지능기능상세
4. 접수번호 기준으로 접수대장·인공지능제품모델·인공지능기능상세에 등록, 일정관리에 행 추가
   - **append-only**: 이미 등록된 접수번호가 다시 들어오면 세 시트 모두 기존 행을 그대로 두고 건너뜁니다(파싱로그에 `건너뜀` 기록). 자동 갱신되지 않으므로, 보완자료로 내용이 바뀐 경우 시트를 직접 수정해야 합니다.
5. 신규 접수 시 활성 심사원을 배정순서대로 라운드로빈 초안 배정하되 Chat 알림은 자동 발송하지 않음
6. 관리자가 일정관리에서 담당심사원을 검토·수정한 뒤 행을 선택하고 **📨 선택 행 배정 확정·알림 발송** 실행
7. 선택 건을 담당자별로 묶어 공용 Google Chat 방에 한 번 발송하고 결과를 `배정알림로그`에 기록

### 잘못 등록한 접수 건 삭제

별도의 자동 삭제 기능은 제공하지 않습니다. 관리자가 동일한 `접수번호`를 검색하여 **접수대장·일정관리·인공지능제품모델·인공지능기능상세**에서 해당 접수번호의 모든 행을 직접 삭제합니다. `파싱로그`는 작업 이력으로 유지하고, Drive의 접수번호 폴더는 필요 여부를 확인한 뒤 별도로 정리합니다.

### 컬럼매핑 (코드 수정 없이 헤더 대응)

KOSA 엑셀의 실제 컬럼명이 바뀌면 **`컬럼매핑` 시트**의 별칭 칸만 수정하면 됩니다. (A열=내부키, B열=쉼표 구분 별칭). 파서는 완전일치 → 부분일치 순으로 매칭합니다.

## 심사 흐름

1. 일정관리(또는 접수대장) 시트에서 대상 행을 선택
2. 메뉴 → **📑 기술심사보고서 생성 (선택 행)** 실행
3. Google Docs 보고서 자동 생성 (접수번호별 Drive 폴더에 저장) 후 문서 URL 알림

### 보고서 구성

1. 심사 개요
2. 신청기업 현황
3. 심사 대상 제품
4. 핵심 인공지능 기능 명세
5. 종합 심사 의견 (5.1. 심사 항목별 검토 결과 포함)
6. 붙임 1. 기능별 인공지능 구현 세부 사항
7. 붙임 2. 데이터 구조도
8. 붙임 3. 첨부자료 목록

### 확인 기준 (가이드라인 1.1~3.1)

| 번호 | 구분 | 확인 내용 | 방법 |
|---|---|---|---|
| 1.1 | 인공지능처리 연산체계 | 연산체계가 적용되어 있는가 | 서류 |
| 1.2 | 〃 | 학습·추론 등 AI 특성을 가지는가 | 서류·현장 |
| 1.3 | 〃 | 유의미한 결과를 출력하는가 | 서류·현장 |
| 2.1 | 인공지능기능 | 구조도상 위치가 명확하고 결합 동작하는가 | 서류 |
| 2.2 | 〃 | 기능·효율성 등에 활용됨이 확인 가능한가 | 서류·현장 |
| 2.3 | 〃 | 주요 기능이 정상 동작하는가 | 현장 |
| 3.1 | 외부 연동 | 호출 로그·API Key 확인 가능한가 | 서류·현장 |

### 구현방식 분류 (인공지능기능상세 시트의 `구현방식`)

`A` 자체 학습 모델 · `B` 오픈소스 모델(사전학습+파인튜닝) · `C` 외부 AI API 연동 · `D` 온디바이스 AI · `E` 혼합형 AI 구성

각 코드별로 요구되는 전용 필드가 다릅니다 (자세한 내용은 [입력폼_명세서.md](./입력폼_명세서.md) 참고).

### 상태값

- **접수대장 `상태`**: 접수 · 심사중 · 보완요청 · 완료 · 반려
- **접수대장 `종합판정`**: 적합 · 보완요청 · 부적합(미충족)
- **일정관리 `상태`**: 대기(기본값) · 심사중 · 보완 · 완료(적합) · 종료(부적합)

## 데이터 무결성 설계

- **모든 시트 쓰기는 헤더 이름 기반** (위치 기반 아님). 심사원이 컬럼을 수동 추가·삽입해도 값이 밀리지 않고, 모르는 컬럼은 빈칸 유지.
- **마이그레이션**: 구버전 시트명·컬럼명 자동 이관 (`헤더마이그레이션`). 이름이 바뀐 컬럼은 기존 값도 새 컬럼으로 복사.
- 위험 작업(시트 재생성 등)은 관리자 메뉴로만 노출.
