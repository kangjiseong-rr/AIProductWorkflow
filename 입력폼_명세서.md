# 입력폼(신청서) 항목 명세서

KOSA(확인기관) 신청 시스템에서 접수되어 TTA로 **엑셀 인수**되는 입력폼 항목의 명세입니다.
KOSA 엑셀은 3개 탭(접수현황 / 제품모델 / 인공지능기능상세)으로 구성되며, `엑셀파싱등록` 실행 시 각각
**접수대장**, **인공지능제품모델**, **인공지능기능상세** 시트에 접수번호 기준으로 등록됩니다.

- 예시 값은 `샘플_신청데이터_10건.xlsx` / `샘플_신청데이터_통합유형_1건.xlsx`의 실제 샘플 데이터를 인용했습니다.
- "필수 여부"는 신청서 상 필수 입력 여부이며, "조건부 필수"는 다른 항목의 선택값에 따라 필수가 되는 항목입니다.
- 컬럼명은 TTA 시트의 내부 헤더명입니다. KOSA 엑셀의 실제 헤더 표기가 다르더라도 **컬럼매핑** 시트의 별칭으로 자동 인식됩니다.

| 대표자 | 문자열 | 대표자 성명 | `홍길동` | 필수 | |

| 제공형태 | 열거형 | 제품·서비스 제공 형태 | `SaaS(클라우드 서비스형)` | 필수 | 그 외 `사내 구축형(온프레미스)` 등. 코드값은 KOSA 신청 시스템 정의 |

| 항목명 | 데이터타입 | 설명 | 입력 예시 | 필수 여부 | 비고 |

## 3. 인공지능기능상세 (→ 인공지능기능상세 시트)

접수번호당 기능 단위 반복행(1건 = N행) 구조입니다. 공통 항목 + `구현방식` 선택값(A~E)에 따른 전용 항목으로 구성되며,
실제로는 구현방식과 무관하게 모든 하위 필드를 한 행에 함께 갖고 있다가(선택되지 않은 유형의 칸은 공란) 보고서에서만 구현방식별로 골라 표시합니다.

| 항목명 | 데이터타입 | 설명 | 입력 예시 | 필수 여부 | 비고 |
|---|---|---|---|---|---|
| 접수번호 | 문자열(FK) | 소속 신청 건 | `AI202600001` | 필수 | |
| 기능번호 | 숫자 | 같은 접수번호 내 기능 순번 (1부터) | `1` | 필수 | 접수대장 "인공지능기능수"는 이 개수로 자동 계산 |
| 기능명 | 문자열 | 인공지능 기능명 | `문서 자동분류` | 필수 | 접수대장 "인공지능기능명(요약)"에 자동 합산 표시 |
| 인공지능역할 | 장문 텍스트 | 해당 기능에서 인공지능이 수행하는 역할 | `업로드된 문서의 본문을 분석하여 문서 유형을 분류…` | 필수 | |
| 입력 | 문자열 | 기능에 입력되는 데이터 종류(요약) | `PDF 문서, 텍스트 본문, 문서 제목…` | 필수 | |
| 출력 | 문자열 | 기능이 출력하는 결과 종류(요약) | `문서 유형, 추천 카테고리, 분류 결과, 신뢰도 점수` | 필수 | |
| 레퍼런스참조위치 | 문자열 | 제출 설명서·매뉴얼 내 해당 기능 설명 위치 | `사용자 매뉴얼 p.12~15` | 필수 | |

## 4. 참고 — TTA 내부 관리 항목 (신청폼에는 없음)

아래 항목은 KOSA 신청폼에서 오는 값이 아니라, TTA가 접수 이후 **자동 계산**하거나 **심사 과정에서 직접 입력**하는 항목입니다.

| 항목명 | 소속 시트 | 데이터타입 | 채우는 방식 | 설명 |
|---|---|---|---|---|
| 심사접수일 | 접수대장 | 날짜 | 자동 | TTA가 KOSA로부터 건을 인수(파싱)한 날짜 |
| 상태 | 접수대장 | 열거형 | 직접 입력 | `접수` · `심사중` · `보완요청` · `완료` · `반려` |
| 제품명 | 접수대장 | 문자열 | 자동 | 인공지능제품모델 시트의 모델명들을 합쳐 표시 |
| 제품수 | 접수대장 | 숫자 | 자동 | 인공지능제품모델 시트의 행(연번) 개수 |
| 인공지능기능수 | 접수대장 | 숫자 | 자동 | 인공지능기능상세 시트의 행(기능번호) 개수 |
| 인공지능기능명(요약) | 접수대장 | 문자열 | 자동 | 기능명들을 `/`로 합쳐 표시 |
| 구현방식(요약) | 접수대장 | 문자열 | 자동 | 사용된 구현방식(A~E) 코드들을 중복 제거해 표시 |
| 담당심사원 | 접수대장 / 일정관리 | 문자열 | 자동 배분(라운드로빈) 후 직접 수정 가능 | `CONFIG.담당자목록` 순환 배정 |

## 추가로 넣으면 좋을 열 (검토 제안)

현재 6개 열(항목명·데이터타입·설명·예시·필수여부·비고) 외에 다음을 더 넣을지는 운영 방식에 따라 선택하시면 됩니다.

- **글자수/형식 제약**: 사업자번호(000-00-00000), 이메일 형식 등 KOSA 신청 시스템이 실제로 강제하는 입력 검증 규칙이 있다면 명시하면 파싱 오류 대응에 유용합니다.
- **KOSA 화면상 항목명 원문**: 지금 표는 TTA 내부 헤더명 기준입니다. 신청 화면에 보이는 문구(예: "제품 또는 서비스 모델명")를 별도 열로 병기하면 KOSA와의 용어 대조가 쉬워집니다.
- **컬럼매핑 별칭**: `컬럼매핑` 시트에 등록된 대체 헤더명 목록을 항목별로 병기하면, 엑셀 헤더가 바뀌었을 때 이 문서만 보고도 별칭 등록 여부를 바로 확인할 수 있습니다.
- **버전/변경이력**: 신청폼 항목이 개편될 때(고시 개정 등) 이 문서도 함께 갱신되도록 최상단에 "기준일"을 표시하는 방법도 있습니다.
